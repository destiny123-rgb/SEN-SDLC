Student Result Management System

Course: SEN 201

Project Type: SDLC Implementation

1. Requirement Analysis

The Student Result Management System is designed to manage and process student academic results. The system should be able to:

  • Accept student name
  • Accept multiple student scores
  • Calculate total and average score
  • Assign grades based on average score
  • Display the final result clearly

2. System Design

The system is designed using Python programming language. The following functions are used in the design and implementation:

  • get_scores() – collects student scores
  • calculate_average() – calculates total and average
  • assign_grade() – assigns grade based on average
  • display_result() – displays the student result

3. Implementation

The system is implemented using Python. All function names used in the design stage are strictly maintained in the implementation to ensure consistency.

4. Testing

The system was tested using different sets of student scores to ensure correct calculation of total, average, and grade assignment.

5. Deployment

The system is deployed as a Python program and hosted on GitHub for easy access, usage, and submission.

6. Maintenance

Future improvements may include adding more subjects, saving results to a file, or developing a graphical user interface.
